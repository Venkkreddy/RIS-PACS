# Project Rules and Guidelines

- **Git & Documentation Synchronization Routine**:
  - **First Task/Prompt of the Day**: Always start by executing a `git pull origin main` to ensure the local workspace matches the remote repository.
  - **Last Task/Prompt of the Day**: Always update `docs/PROJECT_DOCUMENTATION.md` with descriptions of any new features, API endpoints, or configurations introduced. Once updated, stage all modifications (`git add .`), commit, and execute `git push origin main` to keep the remote repository synchronized.
